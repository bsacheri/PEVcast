# PEVcast Development Instructions

## Version Management - CRITICAL

**Whenever making changes to be committed and pushed to git, ALWAYS update version information across all files:**

### Files to Update:
1. **app.js** (line ~16): `window.APP_VERSION = 'X.X.XX'`
2. **version.json**: Update `"version"`, `"js"`, and `"timestamp"` fields
3. **index.html** (line ~34): Update `FILE_VERSIONS` object with `html`, `css`, `js` versions
4. **About Dialog in app.js** (showAboutDialog function ~941): Uses dynamic `window.APP_VERSION` and `generateCodeUpdateTimestamp()` - these will auto-update if you update app.js version

### Version Numbering Scheme:
- Format: `X.X.XX` (e.g., 7.12.25)
- Patch version (last two digits) increments for bug fixes and small improvements
- Minor version (middle digit) increments for feature additions
- Major version (first digit) increments for significant changes

### Example Commit Flow:
1. Make code changes to app.js, styles.css, index.html, etc.
2. Update `window.APP_VERSION` in app.js to new patch version
3. Update `version.json` with matching version and today's date/time
4. Update `FILE_VERSIONS` in index.html with component versions
5. Commit with message describing changes
6. Push to origin/master

### Verification:
After commit, the About dialog will automatically show:
- Current app version (from window.APP_VERSION)
- Current timestamp (generated when dialog opens)

This ensures all version information stays consistent and users always see accurate version details.
